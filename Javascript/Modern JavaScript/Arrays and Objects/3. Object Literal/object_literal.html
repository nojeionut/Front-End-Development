Object Literals

A very common JavaScript data structure that holds key/value pairs is called an object literal. We can also create objects using a constructor, but the object literal syntax is used most often. The object literal is a singleton, meaning it is a single instance of the object. With a constructor, multiple instances of the object can be created.

Below is an example of the syntax to create a new object:

An object is essentially a set of properties with key/value pairs. The key is often called the property name. 

const person = {
  name: 'John',
  age: 30,
  location: 'Boston',
  hobbies: ['movies', 'music'],
  isAdmin: false,
};

Accessing Property Values

We can use dot notation or bracket notation to access property values:

// Dot notation
person.name; // John

// Bracket notation
person['name']; // John

To access the first hobby in the array:

person.hobbies[0]; // movies

Adding & Updating Properties

To programmatically add or change properties:

person.email = 'john@gmail.com'; // Add a new property
person.isAdmin = true; // Update an existing property

Removing Properties

To remove a property, use the delete keyword:

delete person.isAdmin;

Adding Functions to Objects

We can store functions as properties of an object:

person.greet = function () {
  console.log('Hello, my name is ' + this.name);
};

person.greet(); // Hello, my name is John

Notice the use of the this keyword to refer to the object itself.

Using Multiple Words in Property Names

To use multiple words as a property name, wrap the property name in quotes:

const person2 = {
  'first name': 'Brad',
  'last name': 'Traversy',
};

person2['first name']; // Brad
