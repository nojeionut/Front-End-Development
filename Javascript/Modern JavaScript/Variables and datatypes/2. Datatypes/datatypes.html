Data Types

We're going to spend a little time on data types. When we create a variable, the data that we define has a type. There are primitive types and reference types. The big difference under the hood is the way that they're stored and accessed from memory.

  1. Primitive Types - Stored directly in the location that the variable is accessed
  2. Reference Types (Objects) - Accessed by reference

Primitive Data Types

The 7 primitive data types are the following:

  • String - A sequence of characters.
  • Number - Represents both integer and floating-point numbers.
  • Boolean - Represents true or false.
  • Null - Intentional absence of any object value.
  • Undefined - A variable that hasn't been assigned a value.
  • Symbol - Unique identifier.
  • BigInt - For numbers larger than Number type can handle.

Dynamic vs Static Types

JavaScript is dynamically typed, meaning types are associated with values, not variables. Example:

const y = 100;

In TypeScript:

const y: number = 100;

Assigning Variables

String

Number

const age = 30;
const temp = 98.9;

Boolean

const hasKids = true;

Null

const aptNumber = null;

Undefined

let score;

Symbol

const id = Symbol('id');

BigInt

const n = 9007199254740991n;

typeof Operator

console.log(typeof name);

Reference Data Types (Objects)

Reference types include Arrays, Functions, and Objects. They are accessed by reference.

Arrays

const numbers = [1, 2, 3, 4];

Object Literals

{
  name: 'John',
  age: 30
}

Functions

const sayHello = function() {
  return 'Hello';
};

sayHello();
